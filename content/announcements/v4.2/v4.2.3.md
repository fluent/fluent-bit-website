###### KNOWLEDGE BASE

## Release Notes v4.2.3

Fluent Bit is a Fast and Lightweight Data Processor and Forwarder for Linux,
BSD, Windows and macOS. We are proud to announce the availability of Fluent
Bit v4.2.3.

Below is a detailed list of changes included in this release...


### What's new ?

Fluent Bit v4.2.3 is a maintenance release focused on stability and bug fixes
across multiple plugins. It includes fixes for Splunk and Windows Event Log
input handling, Kubernetes and log-to-metrics filter cleanup paths, OpenTelemetry
output gRPC status handling over HTTP/2, Stackdriver OAuth2 lifecycle cleanup,
and AWS-related updates and test coverage improvements.

<br>

#### Core

<br>

- plugin_proxy: enable `event_type` specification for proxy plugins
  ([a36b225a7](https://github.com/fluent/fluent-bit/commit/a36b225a7))
- aws: additional cleanup and adjustments
  ([091593301](https://github.com/fluent/fluent-bit/commit/091593301))
- processor: properly handle numeric values in config map
  ([527ca9fc9](https://github.com/fluent/fluent-bit/commit/527ca9fc9))
- filter: validate `value` property (`not null`)
  ([a2f64b982](https://github.com/fluent/fluent-bit/commit/a2f64b982))
- config_format: align `cf_yaml` `dirname` behavior with POSIX
  ([54a36fbe0](https://github.com/fluent/fluent-bit/commit/54a36fbe0))
- cmake: kafka: fix OAuth Bearer detection on Windows
  ([9191f7079](https://github.com/fluent/fluent-bit/commit/9191f7079))
- bin: add NULL checks and improve `CONT` signal handling in leaks mode
  ([ab34c0ea3](https://github.com/fluent/fluent-bit/commit/ab34c0ea3),
  [bf74f1da2](https://github.com/fluent/fluent-bit/commit/bf74f1da2))
