###### KNOWLEDGE BASE

## Release Notes v5.0.2

Fluent Bit is a Fast and Lightweight Data Processor and Forwarder for Linux, BSD, Windows and macOS. We are proud to announce the availability of Fluent Bit v5.0.2.

Below is a detailed list of changes included in this release...


### What's new ?

<br>

#### Core

<br>

- opentelemetry: add OTLP JSON and protobuf encoders
- pack: add `otlp_json`, `otlp_json_pretty`, and `otlp_proto` formats
- oauth2: harden token refresh parsing ([PR #11618](https://github.com/fluent/fluent-bit/pull/11618))
- aws: compression: arrow: set up proper block size ([PR #11640](https://github.com/fluent/fluent-bit/pull/11640))

### Plugins

<br>

- **[Azure Blob (Output)](https://docs.fluentbit.io/manual/pipeline/outputs/azure_blob)**
  - add path templating support comparable to S3 ([PR #11178](https://github.com/fluent/fluent-bit/pull/11178))
- **[Azure Kusto (Output)](https://docs.fluentbit.io/manual/pipeline/outputs/azure_kusto)**
  - improve stability with cleanup fixes and better HTTP buffer sizing ([PR #11607](https://github.com/fluent/fluent-bit/pull/11607))
- **[eBPF (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/ebpf)**
  - add VFS trace support ([PR #11568](https://github.com/fluent/fluent-bit/pull/11568))
- **[Kafka (Output)](https://docs.fluentbit.io/manual/pipeline/outputs/kafka)**
  - add `otlp_json` output format so logs, metrics, and traces can be emitted as OTLP JSON
- **[Stdout (Output)](https://docs.fluentbit.io/manual/pipeline/outputs/standard-output)**
  - add `otlp_json` and `otlp_json_pretty` print formats
