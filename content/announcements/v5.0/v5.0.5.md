###### KNOWLEDGE BASE

## Release Notes v5.0.5

Fluent Bit is a Fast and Lightweight Data Processor and Forwarder for Linux, BSD, Windows and macOS. We are proud to announce the availability of Fluent Bit v5.0.5.

This maintenance release improves OTLP log grouping and routing correctness, adds new S3 and eBPF capabilities, and includes reliability improvements across AWS, Kubernetes, LogDNA, JSON handling, metrics, and build compatibility.


### What's new ?

<br>

#### Core

<br>

- opentelemetry: improve OTLP log resource and scope preservation across ingestion, routing, and re-export
- routing: enable conditional routing for threaded inputs and improve route preservation when records are split across matched and default routes
- json: improve mutable JSON handling reliability for complex documents
- aws: expand endpoint coverage for additional AWS region families
- crypto: improve compatibility with OpenSSL builds without legacy engine support
- tests: expand integration and runtime coverage for OTLP grouping, conditional routing, Forward metadata defaults, S3 OTLP JSON, LogDNA formatting, eBPF exec tracing, AWS regions, Kubernetes cleanup, and Log to Metrics timer retries

#### Libraries

<br>

- cmetrics: upgrade the bundled library to v2.1.3 with improved OTLP metric compatibility and decoding reliability

### Plugins

<br>

- **[eBPF (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/ebpf)**
  - add an `exec` trace for process execution observability
- **[OpenTelemetry (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/opentelemetry)**
  - improve OTLP log ingestion reliability and preserve scope metadata when converting logs into Fluent Bit events
- **[S3 (Output)](https://docs.fluentbit.io/manual/pipeline/outputs/s3)**
  - add `format otlp_json` for writing log events to S3 as OTLP JSON, while keeping `json_lines` as the default format
  - improve validation for OTLP JSON output options
- **[OpenTelemetry (Output)](https://docs.fluentbit.io/manual/pipeline/outputs/opentelemetry)**
  - improve grouped log reassembly while preserving resource and scope identity
  - improve metadata validation and OAuth2 cleanup during shutdown
- **[Forward (Output)](https://docs.fluentbit.io/manual/pipeline/outputs/forward)**
  - retain metadata in forward mode by default through `retain_metadata_in_forward_mode true`
- **[LogDNA (Output)](https://docs.fluentbit.io/manual/pipeline/outputs/logdna)**
  - add `exclude_promoted_keys` to omit promoted `meta`, `level` or `severity`, `app`, and `file` fields from the `line` body while preserving non-promoted fields
- **[Kubernetes (Filter)](https://docs.fluentbit.io/manual/pipeline/filters/kubernetes)**
  - improve token handling and AWS pod association resource cleanup
- **[Log to Metrics (Filter)](https://docs.fluentbit.io/manual/pipeline/filters/log_to_metrics)**
  - improve timer-driven retry behavior for metrics emission
- **[Node Exporter Metrics (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/node-exporter-metrics)**
  - handle unavailable IPv6 socket metrics cleanly
